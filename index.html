System Overview

CareHive is an integrated AI-powered monitoring ecosystem consisting of three primary hardware components working in synergy with a cloud-based analytical platform. The system enables continuous, non-intrusive monitoring of elderly residents in care facilities through advanced sensor fusion and real-time data processing.

Wearable Device Technical Specification

Hardware Design

The CareHive wearable is designed as a smart ring form factor, optimized for continuous wear comfort and medical-grade monitoring accuracy. The device utilizes a custom PCB design with advanced sensor integration.

Form Factor Rationale

Why Ring Over Bracelet: Ring form factor provides superior PPG signal quality due to consistent contact pressure, reduced motion artifacts, and higher user compliance (92% vs 67% for wrist-worn devices in elderly populations). Additionally, rings are less likely to be removed during daily activities.

UWB Anchor System

System Architecture

The anchor network provides centimeter-level indoor positioning using Ultra-Wideband (UWB) technology. Anchors are strategically placed throughout the facility to create overlapping coverage zones, enabling continuous 3D position tracking with <10cm accuracy.

Why UWB Over WiFi/BLE Positioning?

Accuracy: UWB provides 5-10cm accuracy vs 1-5m for WiFi/BLE triangulation. Latency: <50ms update rate vs 1-3s for signal strength methods. Penetration: UWB signals are less affected by walls and obstacles. Multi-path Resistance: Time-of-flight measurement is immune to reflections that plague RSSI-based systems.

Positioning Algorithm

Time Difference of Arrival (TDoA)

// Multilateration using TDoA measurements
$d_i = c \times (ToA_i - ToA_{reference})$

// Non-linear least squares optimization
$$\min \sum_{i} \left[\sqrt{(x-x_i)^2 + (y-y_i)^2 + (z-z_i)^2} - d_i\right]^2$$

// Kalman filter for position smoothing
$\mathbf{x}_{estimated} = \mathbf{x}_{predicted} + \mathbf{K} \times (\mathbf{z}_{measured} - \mathbf{H} \times \mathbf{x}_{predicted})$

where:
$c = 3 \times 10^8\ m/s$ (speed of light)
$\mathbf{K} =$ Kalman gain matrix
$\mathbf{H} =$ observation model

Coverage Optimization

Positioning accuracy degrades with poor anchor geometry. Optimal configuration requires:

  • **Geometric Dilution of Precision (GDOP) < 3:** Ensures sub-10cm accuracy
  • **Minimum 4 anchors per zone:** Required for 3D positioning (3D trilateration + clock offset)
  • **Anchor spacing:** 8-15m optimal, <25m maximum
  • **Height variation:** Mix of ceiling-mounted (2.7m) and wall-mounted (1.8m) for vertical accuracy

Placement Optimization & Infrastructure

Anchor Network Planning

The placement strategy is critical to achieving the required sub-10cm accuracy across the facility. It balances signal integrity, hardware minimization, and installation aesthetics.

  • **Coverage Metric:** The system targets a minimum GDOP (Geometric Dilution of Precision) value of 3.0 or less across 98% of all resident-accessible areas.
  • **Channel Allocation:** UWB channels 5 and 9 are utilized to minimize interference and maximize bandwidth for time-of-flight measurements.
  • **Cable Infrastructure:** PoE (Power over Ethernet) is the mandatory powering and primary communication protocol for all Anchors to ensure reliability and simplify maintenance.

Wireless Network Requirements

The facility must provide a robust, segregated network for CareHive data transfer, ideally meeting the following minimum specifications:

  • **Bandwidth:** Minimum 100 Mbps uplink/downlink for the central gateway.
  • **Network Segregation:** Dedicated VLAN for all CareHive devices (HIPAA/GDPR compliance).
  • **Gateway:** Centralized server (local or edge) for data aggregation before cloud transmission. Specs: 16-core CPU, 64GB RAM, 10TB SSD.

  • **Monitoring:** Prometheus & Grafana stack for system health monitoring.

AI/ML Pipeline

The predictive health and anomaly detection engine relies on a multi-stage ML process:

  1. **Data Pre-processing:** Noise filtering (Kalman), normalization, and feature extraction (e.g., Fourier Transform on PPG).
  2. **Baseline Model Training:** Train patient-specific models on 7 days of initial data to establish a 'normal' behavioral and physiological baseline.
  3. **Anomaly Detection:** Real-time classification using an Isolation Forest or Autoencoder model trained on the patient's baseline.
  4. **Alert Scoring:** Final risk score generated by a weighted combination of: Positioning Anomaly (e.g., long bathroom time), Physiological Deviation (e.g., Bradycardia), and Environmental Context (e.g., low temperature).
